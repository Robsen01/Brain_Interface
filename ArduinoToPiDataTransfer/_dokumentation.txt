Datenübertragung funktioniert über den USB mit der Serial Schnittstelle.
Im arduino mit Serial.read() oder Serial.print(). Im PC mit dem pyserial modul. pip install pyserial
Tutorial: https://create.arduino.cc/projecthub/ansh2919/serial-communication-between-python-and-arduino-e7cce0

Auf Python Seite liest ein Thread die ganze zeit die eintreffenden Daten und schreibt sie in eine Queue/Buffer
Auf anderen Threads, z.B. auf den der GUI können Daten zum Arduino geschickt werden. Z.B. welche Threashhold für den Filter gewollt ist.
Das ist allerdings nur nötig, wenn wir auf dem Arduino überhaupt daten verarbeiten wollen.

Auf Arduino Seite ist "echtes" multithreading nicht möglich und in unserem Fall auch nicht nötig. Hier werden zu erst die Sensordaten gelesen, dann verschickt, dann werden eingetroffene Serial daten gelesen und verarbeitet. Dann fängt ein neuer Cycle an.
